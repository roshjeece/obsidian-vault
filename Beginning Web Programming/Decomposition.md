Algorithms

- Everything we do with coding involves problem solving. The way we solve these problems are with algorithms. Sometimes referred to as “algos”
- What are some questions you need to ask?
- What are some strategies?
- **The best way to solve a problem is to decompose it**

How do we take a problem and break it down to smaller parts?

- Think about painting a wall. Do you just wake up and start painting?
- What steps do you take to get the wall painted
- Decide on a color:
	- Research:
		- Online,
		- At a paint store,
		- Recommendations
- Buy supplies
	- Paint brush
	- Paint pan
	- Painter's tape
	- Paint cleaner
	- Drop cloth
	- Paint
- Prep the wall
	- Tape edges
	- Cover windows
	- Door knobs, hinges, light switches
- Prepare the paint
	- Shake, stir, pour into pan
- Paint the wall
	- Edges first?
	- Roller in the middle?
	- Corners first?
- Examine work
	- Looks good - clean up
	- Else - touch up and second coat, repeat painting the wall
- Clean Up
	- Wall
		- Remove drop cloth
		- Remove tape
	- Paint supplies
		- Clean brushes
		- Clean paint pan
		- Wash out rollers

Pseudo Code
- We can also use pseudo code in addition to decomposition
- This can be written in many ways
- Plain language description of the steps in an algorithm or another system
- This is "code" written for humans to understand, not computers
- This "code" can be written in many ways, whatever I understand best
- Example:
	- *How do we print out all numbers between -10 and 10 but only if they are NOT multiples of 3?*
	- What questions do you have?
	- What might your strategy be?
	- How would we write some pseudo code to write this algorithm?
