# Spring Boot - How to Build This Unassisted

BLUF: You can stand up a new Spring Boot REST API from scratch by following a repeatable sequence. This note is the checklist and mental model — everything you need to go from zero to a working, tested endpoint without looking anything up.

---

## The Mental Model

Every Spring Boot feature you build follows this pattern:

```
Database schema (SQL migration)
        ↓
Entity (Java class that maps to the table)
        ↓
Repository (interface that queries the entity)
        ↓
Service (business logic that calls the repository)
        ↓
Controller (HTTP endpoints that call the service)
        ↓
Tests (written before or alongside each layer)
```

You always go bottom-up (database first) when building new features. You always go top-down (controller first) when debugging.

## Step 1: Create the Project

Go to [start.spring.io](https://start.spring.io) and select:

| Setting | Value |
|---|---|
| Project | Gradle - Groovy |
| Language | Java |
| Spring Boot | Latest stable |
| Java | 21 |
| Dependencies | Spring Web, Spring Data JPA, PostgreSQL Driver, Flyway Migration, Docker Compose Support |

Download, unzip, open in IntelliJ.

## Step 2: Configure the Project

**`compose.yaml`** — your local database:
```yaml
services:
  postgres-db:
    container_name: my_app
    image: postgres:18
    environment:
      POSTGRES_USER: app_dev
      POSTGRES_PASSWORD: app_dev_pw
      POSTGRES_DB: my_app
    ports:
      - "5434:5432"
```

**`application.yaml`** — connect the app to the database:
```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5434/my_app
    driver-class-name: org.postgresql.Driver
    username: app_dev
    password: app_dev_pw
  jpa:
    hibernate:
      ddl-auto: validate    # let Flyway own the schema, not Hibernate
    show-sql: true
  flyway:
    enabled: true
    locations: classpath:db/migration
server:
  port: 8080
```

## Step 3: Write the Database Migration

Create `src/main/resources/db/migration/V1__init.sql`:

```sql
CREATE TABLE thing (
    id    BIGINT GENERATED BY DEFAULT AS IDENTITY NOT NULL,
    name  TEXT,
    CONSTRAINT pk_thing PRIMARY KEY (id)
);
```

**Flyway naming rules:**
- Must start with `V` (capital)
- Version number (e.g. `1`, `2`, `1_1`)
- Double underscore `__`
- Descriptive name
- `.sql` extension

Next migration: `V2__add_column_to_thing.sql`. Never edit a migration that has already run.

## Step 4: Create the Entity

```java
@Entity
@Table(name = "thing")
public class Thing {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;

    public Thing() {}           // required by JPA
    public Thing(String name) { this.name = name; }

    // getters and setters for every field
}
```

**Checklist:**
- `@Entity` on the class
- `@Id` + `@GeneratedValue` on the primary key
- No-arg constructor
- Getters and setters for all fields
- Field names match column names (snake_case conversion is automatic)

## Step 5: Create the Repository

```java
@Repository
public interface ThingRepository extends JpaRepository<Thing, Long> {
    // Free: save, findById, findAll, deleteById
    // Add custom queries here as needed:
    Optional<Thing> findThingByName(String name);
}
```

---

## Step 6: Create the Service

```java
@Service
public class ThingService {

    private final ThingRepository thingRepository;

    public ThingService(ThingRepository thingRepository) {
        this.thingRepository = thingRepository;
    }

    public Thing save(Thing thing) {
        return thingRepository.save(thing);
    }

    public List<Thing> findAll() {
        return thingRepository.findAll();
    }

    public Thing findById(Long id) {
        return thingRepository.findById(id)
            .orElseThrow(() -> new ResponseStatusException(HttpStatus.NOT_FOUND));
    }
}
```

---

## Step 7: Create the Controller

```java
@RestController
@RequestMapping("/api/v1/thing")
public class ThingController {

    private final ThingService thingService;

    public ThingController(ThingService thingService) {
        this.thingService = thingService;
    }

    @PostMapping
    @ResponseStatus(HttpStatus.CREATED)
    public Thing save(@RequestBody Thing thing) {
        return thingService.save(thing);
    }

    @GetMapping
    public List<Thing> findAll() {
        return thingService.findAll();
    }

    @GetMapping("/{id}")
    public Thing findById(@PathVariable Long id) {
        return thingService.findById(id);
    }
}
```

## Step 8: Write Tests (TDD — write these before the implementation)

**Service test:**
```java
@ExtendWith(MockitoExtension.class)
class ThingServiceTest {
    @Mock ThingRepository thingRepository;
    @InjectMocks ThingService thingService;

    @Test
    void shouldSaveThing() {
        Thing thing = new Thing("widget");
        when(thingRepository.save(thing)).thenReturn(thing);
        Thing result = thingService.save(thing);
        assertThat(result.getName()).isEqualTo("widget");
        verify(thingRepository, only()).save(thing);
    }
}
```

**Controller test:**
```java
@WebMvcTest(ThingController.class)
class ThingControllerTest {
    @Autowired MockMvc mockMvc;
    @Autowired ObjectMapper objectMapper;
    @MockitoBean ThingService thingService;

    @Test
    void shouldSaveThing() throws Exception {
        Thing thing = new Thing("widget");
        thing.setId(1L);
        when(thingService.save(any())).thenReturn(thing);

        mockMvc.perform(post("/api/v1/thing")
                .contentType(MediaType.APPLICATION_JSON)
                .content(objectMapper.writeValueAsString(thing)))
            .andExpect(status().isCreated())
            .andExpect(jsonPath("$.name").value("widget"));
    }
}
```

**Repository test:**
```java
@DataJpaTest
class ThingRepositoryTest {
    @Autowired ThingRepository thingRepository;

    @Test
    void shouldSaveThing() {
        Thing thing = new Thing("widget");
        thingRepository.save(thing);
        Optional<Thing> result = thingRepository.findById(thing.getId());
        assertThat(result.get().getName()).isEqualTo("widget");
    }
}
```

## Common Mistakes and How to Avoid Them

| Mistake | Fix |
|---|---|
| Forgetting the no-arg constructor on an entity | JPA requires it — always include it |
| Editing a Flyway migration that already ran | Create a new V2__ file instead |
| Putting business logic in the controller | Move it to the service |
| Using `@Autowired` on fields instead of constructor injection | Use constructor injection — fields should be `final` |
| `ddl-auto: update` with Flyway | Use `ddl-auto: validate` — let Flyway own the schema |
| Not handling `Optional` — calling `.get()` without checking | Use `orElseThrow()` or check `isPresent()` first |
| Returning `null` from a controller method | Return an empty list `List.of()` or a `ResponseEntity.noContent()` |

---

## The Order When You're Stuck

When something breaks, walk the chain in this order:

1. **Did the app start?** → Check Flyway logs. Schema mismatch will kill startup.
2. **Is the endpoint reachable?** → Check `@RequestMapping` and `@GetMapping`/`@PostMapping` paths.
3. **Is JSON deserializing?** → Add `.andDo(print())` to your MockMvc test to see what Spring sees.
4. **Is the service being called?** → Add `System.out.println()` or check with a service test.
5. **Is the database query correct?** → Enable `show-sql: true` and read the generated SQL in the console.
6. **Is data actually in the database?** → Connect with a DB client (IntelliJ Database tab, psql) and query directly.
