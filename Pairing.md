### What is Pairing?
- An extreme programming (XP) technique where two members within a balanced team work together to accomplish a task
- Why?
	- Code QA
	- Shared understanding
	- Learning
	- Increased viewpoints
- What does pairing look like in the wild?

### Types of Pairing
- Discipline Pairing
	- Pairing within your discipline (PD with PD, PM with PM, etc)
	- Two people work on the same problem, at the same time, on the same machine (or virtual space)

### Challenges

Remote Pairing

### Techniques to Try
- Ping Pong (dev focused)
- Pomodoro
- Task List
- Tour Guide

#### Ping Pong (Dev Focused)
- What it is:
	- Two developers alternate roles frequently between writing tests and writing code to make tests pass
- How it works:
	- Dev A writes a failing test
	- Dev B writes the code to pass the test
	- Dev B writes the next failing test
	- Dev A writes the code to make that pass
	- and so on
- Goal:
	- Promote TDD and keep both devs engaged

#### Pomodoro
- What it is:
	- 25 minutes work, 5 min break
	- Use it in conjunction with another method

#### Task List
- What it is:
	- A pairing session guided by a clear list of subtasks
- How it works:
	- Break down a story/ticket into smaller, manageable steps
	- Work through each task together
	- Helps keep the session structured and focused
- Goal:
	- Keep momentum and make steady progress without getting lost

#### Driver and Navigator
- What it is:
	- Classic pairing technique inspired by TDD and XP
- How it works:
	- Driver: at the keyboard, driving code
	- Navigator: guides the direction, thinks ahead, spots problems, reviews code in real time
	- Switch roles regularly
- Goal:
	- Divide thinking and doing - helps maintain quality, avoid tunnel vision, improve communication
